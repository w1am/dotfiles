# Agent instructions

This repo is a **chezmoi source directory** (`~/.local/share/chezmoi`) that manages
the user's dotfiles. These are operating instructions for an AI agent making changes
here. `CLAUDE.md` and `README.md` are symlinks to this file.

## Hard rules

- **Never commit secrets.** SSH private keys, tokens, and credentials must not enter
  this repo â€” not even encrypted, unless the user explicitly opts into chezmoi's
  secret/age support. SSH keys are per-machine identity, generated at bootstrap.
- **Edit the source, never the target.** Change `dot_zshrc` here, not `~/.zshrc`.
  Targets are generated by `chezmoi apply`. If a live dotfile was edited directly,
  run `chezmoi add <path>` to pull it back into source before doing anything else.
- **Filename prefixes are semantics, not decoration.** `dot_` â†’ `.`, `private_` â†’
  mode 0700, `run_once_` â†’ run once per machine, `run_onchange_` â†’ re-run when the
  script body changes. Renaming changes behavior. Preserve them deliberately.
- **Anything not in `.chezmoiignore` becomes a `$HOME` file.** Docs (`AGENTS.md`,
  `README.md`, `CLAUDE.md`) are ignored. Any new repo-only file (notes, CI, etc.)
  MUST be added to `.chezmoiignore` or chezmoi will install it into the home dir.
- **`sudo` is not passwordless.** `apt` and `chsh` prompt for a password, so they
  cannot be run non-interactively. Put such steps in bootstrap scripts (which run
  interactively during `chezmoi apply`) rather than executing them yourself.

## Make-a-change workflow

1. Edit the source file (`dot_*`, a script in `.chezmoiscripts/`, or `*.toml`).
2. `chezmoi diff` to preview the effect on the target system.
3. `chezmoi apply` to write changes to the live system.
4. Verify (see below).
5. Commit and push: the user's `update` shell function does
   `chezmoi git -- add -A && commit && push`. Otherwise `chezmoi cd` then git.

## Verification (run before claiming done)

- `chezmoi verify` â€” source and target agree.
- `chezmoi managed | grep <name>` â€” confirm a file is managed / NOT managed as expected.
- For zsh: `zsh -n ~/.zshrc`. For scripts: `sh -n <script>`. For tmux: start a
  detached session and confirm it parses.
- For "is this leaking into `$HOME`?" checks: `chezmoi managed` should NOT list docs.

## Conventions

- **Bootstrap scripts** live in `.chezmoiscripts/`, named `<type>_after_<NN>-<desc>.sh`,
  run in ascending `NN` order. Use `run_onchange_` for things that should re-run when
  their definition changes (package lists, installed-tool versions); use `run_once_`
  for one-time per-machine actions (chsh, key generation). Keep every script
  idempotent and a no-op on non-apt systems.
- **Packages** are declared in the `packages=` list in the install-packages script.
  Detect installation with `dpkg -s` (package name), not `command -v` (command name
  can differ, e.g. `wl-clipboard` provides `wl-copy`).
- **Tools that lag in apt** (e.g. Neovim) are installed from upstream releases into
  `~/.local` (no sudo; `~/.local/bin` is on `PATH`). Pin/track version via a variable
  at the top of the script.
- **Commit messages**: imperative subject line; explain *why* in the body for
  non-obvious changes.

- `dot_tmux.conf` â†’ `~/.tmux.conf` (backtick prefix, vi mode, `wl-copy` clipboard)

- `.chezmoiexternal.toml` â†’ clones Oh My Zsh + zsh plugins
- `.chezmoiscripts/` â†’ install-packages, install-neovim, set-default-shell, generate-ssh-key
- `.chezmoiignore` â†’ keeps the three doc files out of `$HOME`
